# Strategy Engine Reconciliation

## Authoritative runtime position
The modern intelligence runtime under `app/intelligence/` is the target architecture. The legacy `app/services/strategy_engine/` remains authoritative only for capabilities not yet migrated.

## Legacy engine strengths
- richer diagnostic scenario library
- executive summary and strategic score packaging
- temporal and competitive diagnostics modules
- automation phase logic tied to recommendation lifecycle governance

## Modern intelligence strengths
- persistent signal, feature, pattern, outcome, graph, industry, and model layers
- digital twin selection in the orchestrator path
- governed mutation execution and rollback
- durable event stream and distributed worker integration
- cross-campaign transfer, strategy evolution, flight recorder, and network learning

## Unification rule
Do not delete legacy modules until the modern runtime has equivalent diagnostic coverage, scoring outputs, and automation phase handling.

## Migration target flow
```text
signal ingestion
  -> feature engineering
  -> diagnostic and pattern detection
  -> policy and strategy generation
  -> strategy transfer
  -> digital twin simulation
  -> strategy selection
  -> execution planning
  -> governed execution
  -> outcomes and learning
```

## Keep / merge / replace guidance
- Keep modern execution, digital twin, graph, industry, network, and persistence layers.
- Migrate legacy diagnostic modules into modern pattern and recommendation generation.
- Migrate legacy executive summary and strategic score packaging into the modern orchestrator response layer.
- Replace legacy direct recommendation ranking with the modern policy plus digital twin selection path.
