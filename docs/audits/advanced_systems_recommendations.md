# Advanced Systems Recommendations

Date: 2026-03-11

## Purpose

This document recommends where to deepen the platform into more advanced systems and where not to overinvest yet.

## Rule Of Thumb

Advance systems in this order:

1. user-visible leverage
2. workflow leverage
3. data/learning leverage

Do not invert that order.

## Priority Advanced Systems

### 1. Unified campaign operating system

Build:

- campaign home
- health scorecards
- action feed
- ranking, local, technical, competitor, and reporting summaries in one place

Why:

- this turns many isolated APIs into a coherent product

Advanced traits:

- role-based views
- owner vs agency modes
- summary cards + drilldowns

### 2. Local market visibility engine

Build:

- geo-grid / service-area visibility map
- local pack share analysis
- multi-location comparison
- review trend + visibility correlation

Why:

- this is one of the best opportunities to create a category-defining surface

Advanced traits:

- neighborhood/zip/service-area segmentation
- competitor overlay
- time-based playback

### 3. Technical issue prioritization system

Build:

- issue scoring by business impact
- URL issue matrix
- remediation queue
- verification loop after fixes

Why:

- most audit tools overwhelm users; prioritization and closure create product value

Advanced traits:

- confidence by issue type
- page-value weighting
- fix verification state

### 4. Competitor strategy system

Build:

- competitor overlap views
- gap clusters
- relative visibility by theme/service
- competitor-driven strategy prompts

Why:

- current competitor support is real but shallow

Advanced traits:

- intent cluster comparisons
- service-area competitor maps
- recommendation traceability back to competitor evidence

### 5. Recommendation and execution control center

Build:

- action queue
- confidence/risk display
- approval workflows
- execution timeline
- outcome comparison

Why:

- the backend already supports unusually strong execution governance

Advanced traits:

- expected impact ranges
- rollback readiness
- action sequencing

### 6. Reporting narrative engine

Build:

- owner report format
- agency report format
- white-label report format
- evidence and confidence blocks

Why:

- reporting should be one of the product’s strongest commercial assets

Advanced traits:

- narrative templates by audience
- auto-generated monthly highlights
- exception/risk section

### 7. Agency portfolio operating layer

Build:

- cross-account dashboards
- location and subaccount rollups
- alerting and account health matrix
- white-label distribution

Why:

- the backend hierarchy model deserves a stronger portfolio experience

Advanced traits:

- benchmark comparisons
- portfolio prioritization
- manager views and filtered work queues

### 8. Evidence-backed explanation layer

Build:

- evidence viewer
- recommendation "why" cards
- report citations
- summary cards with confidence and source grounding

Why:

- this is the right bridge to a future LLM layer

Advanced traits:

- org-pinned evidence versions
- explanation differences by audience mode

## Systems To Strengthen Before Going More Advanced

### Frontend foundation

Needs:

- real dashboard architecture
- reusable component system
- route-level tests
- stronger visual design language

### Reporting data model integration

Needs:

- richer KPI composition
- clearer business outcome proxies
- stronger connection between actions and results

### Learning architecture cleanup

Needs:

- removal or isolation of duplicated legacy learning paths
- cleaner single-spine story for experiments and learning

## Systems That Should Wait

### More abstract intelligence expansion

Wait until:

- dashboard, reporting, local SEO, and recommendations are much better surfaced

### Heavy future-state AI search abstraction

Wait until:

- entity/AI visibility has a concrete user-facing interpretation

### Broad new feature count expansion

Wait until:

- current breadth is translated into product depth

## Recommended Build Sequence

### Wave 1

- campaign dashboard rebuild
- onboarding UX
- rank and crawl visualizations
- local visibility basics
- report redesign

### Wave 2

- competitor strategy workspace
- recommendation/action center
- technical issue prioritization workspace
- agency rollups

### Wave 3

- evidence-backed explanation layer
- white-label reporting modes
- advanced portfolio controls
- deeper local market comparisons

### Wave 4

- mature LLM summary layer grounded in reference-library evidence
- adaptive guidance by user type and confidence

## Highest-Leverage Moat Builds

If only a few advanced systems are built deeply, prioritize:

1. local market visibility engine
2. recommendation/execution control center
3. reporting narrative engine
4. agency portfolio operating layer
5. evidence-backed explanation layer

## Final Recommendation

The platform should be advanced where the user feels it:

- insight clarity
- visual confidence
- actionability
- trust
- portfolio control

It should not be advanced only where architects feel it.
